# aegis-android

Aegis client for Android. Kotlin + Jetpack Compose + Material 3,
backed by the Rust post-quantum core via
[`mlaify/aegis-ffi`](https://github.com/mlaify/aegis-ffi).

The same Rust crate that powers the relay, gateway, and CLI also
drives this client — no second implementation of ML-KEM-768 /
ML-DSA-65 / hybrid signing.

## Quick start

You'll need:

- JDK 17 (Temurin or Homebrew openjdk@17).
- [`rustup`](https://rustup.rs) with the Android targets:
  ```
  rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android
  ```
- `cargo-ndk`: `cargo install cargo-ndk`
- An Android NDK installation (e.g. via Android Studio → SDK Manager,
  or `sdkmanager 'ndk;27.2.12479018'`). Set `ANDROID_NDK_HOME` if the
  build script can't auto-detect it.
- The sibling repos cloned next to this one:
  ```
  parent/
  ├── aegis-core/
  ├── aegis-ffi/
  └── aegis-android/   ← you are here
  ```

Build the native bridge once after cloning (creates
`app/src/main/jniLibs/{abi}/libaegis_ffi.so` for each Android ABI plus
a host build at `../aegis-ffi/target/release/` for JVM unit tests):

```bash
./scripts/build-jni.sh
```

Then the usual Gradle commands:

```bash
./gradlew :app:test            # 5 JUnit tests through the FFI
./gradlew :app:assembleDebug   # debug APK
./gradlew :app:lintDebug       # Android lint
```

Open in Android Studio for development with full IDE support.

## Why native instead of React Native

The original mobile-track plan used React Native. That was traded in
for native Kotlin (and native Swift on Apple) so:

- Compose / SwiftUI give us native UX feel without an additional
  bridge layer.
- The crypto bridge (`aegis-ffi`) is already cross-platform via
  UniFFI — Apple consumes it as a Swift package, Android consumes it
  as Kotlin bindings + JNI `.so` libraries. Same Rust source of truth.
- JS-bridge marshaling overhead doesn't apply to crypto-heavy
  operations like ML-KEM encapsulation or ML-DSA signing.

## Sibling repo dependencies

`aegis-android` consumes `mlaify/aegis-ffi`, which in turn path-deps to
`mlaify/aegis-core`. The expected workspace layout is all four
side-by-side:

```
parent/
├── aegis-core/
├── aegis-ffi/
├── aegis-apple/
└── aegis-android/    ← you are here
```

CI clones the missing siblings automatically (matching the convention
used by every other Rust-consuming repo in the project).
